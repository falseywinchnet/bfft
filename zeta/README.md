# Zeta / DIP field-agent handoff

This folder is self-contained. It uses `../src/detail/bruun_dip_kernel.hpp`
as a trusted real FFT and does **not** revalidate the transform. The implemented
interface contract is only

`reflection + zero padding + DIP convolution + valid cropping == dense Hankel matvec`.

## Executive result

The operator bridge works. The focused convergence closure at
`omega=2, a=1.5` does not identify a zeta-specific compressed phase law.

- Arithmetic Mellin validation: 12/12 pass at 80 digits; worst relative error
  `1.35e-81`.
- DIP/dense Hankel validation: 192/192 pass; worst relative error `6.14e-16`
  (threshold `1e-10`).
- Halmos dilation: symmetric/unitary residuals are at floating-point noise.
- Recursive local CSD: local reconstruction residuals are at floating-point
  noise.
- Focused Fredholm gate: the trace-corrected cusp-fitted Galerkin sequence
  reaches `|log m_1024-log m_512| = 3.75e-9`.
- Operator convergence: the nested-space extended-operator difference falls
  `2.01e-3, 4.93e-4, 1.26e-4, 3.18e-5`; the leading eigenspaces stabilize at
  the same approximately second-order rate.
- Focused refinement: median drift worsens from `0.399` (`32 -> 64`) to `0.437`
  (`64 -> 128`), rather than falling after operator convergence.
- Compression: zeta's 99.9% ranks at `N=16,32,64` are typically `3,4,5`, the
  same range as the Gaussian, Blaschke, random all-pass, and generic smooth
  phase controls. This is not arithmetic specialization.

The stopping rule is now triggered. The result is **Outcome B: fast solver
without distinguished compression**. The phase-law hypothesis and the
zeta-specific compression hypothesis are closed for the tested DIP basis and
cell observables. No broader omega descent is justified by these results.

## Answers to the seven required questions

1. **Does arithmetic `h_omega` reproduce `Theta_omega` independently?**
   Yes at the Mellin-factorization level. `g` is transformed from its defining
   beta integral, and the Dirichlet series generated by `c_omega` is evaluated
   through its Euler product identity; the result is compared only afterward
   with direct `xi` evaluation. All requested points pass below `1e-25`.

2. **Does the finite canonical system reproduce Suzuki's `m_omega(a)`
   construction for `omega>1`?** Yes at the focused closure point
   `omega=2, a=1.5`. A continuous piecewise-linear Galerkin operator is clipped
   exactly along every `u+v=log(n)` cusp. The adaptive exact trace is separated
   using `log det((I+K)/(I-K)) = 2 Tr(K) + sum_j(log((1+lambda_j)/(1-lambda_j))-2lambda_j)`.
   The final corrected delta is `3.75e-9`, below the requested `1e-8` gate.
   This focused result does not retroactively certify the full `(omega,a)` grid
   or the Fredholm boundary-derivative identities.

3. **Does DIP reproduce the Hankel operator without circular/order errors?**
   Yes. The exact reflection/crop identity is documented in `dip_hankel.py`;
   all 192 small/medium trials pass with worst error `6.14e-16`.

4. **Does the lossless dilation have sublinear effective complexity?** The
   tested scalar summaries have slowly growing rank, but so do all controls.
   Because the depth-by-diagonal table itself has only `O(log N)` rows, this
   rank fact alone is not discovery. There is no distinguished zeta advantage.

5. **Do cell laws converge under refinement?** No. At the converged focused
   point the median drift increases from `0.399` to `0.437`, and the maximum
   increases from `0.719` to `0.882`.

6. **What fails first as `omega` decreases?** Refinement coherence and
   zeta-specific compression have already failed above `omega=1`. During the
   descent the contraction remains visible through `omega=0.55`; the first new
   numerical warning is quadrature regularity near integer singularities.
   No ordinary determinant claim is made at or below `omega=1/2`.

7. **Is the zeta phase tree distinguishable from controls?** Not in these
   observables/resolutions. At `N=128`, zeta's 99.9% effective rank is `6`; all
   four controls have rank `5`. Zeta is not merely within the control range—it
   is slightly less compressible by this measure. The safe conclusion is
   universal Bruun/CSD geometry rather than arithmetic zeta structure.

## Numerical design

`suzuki_kernel.py` supplies `xi`, direct `Theta`, `c`, `g`, `h`, and `k` with
double and high-precision reference paths. `nystrom_operator.py` includes both
composite Gauss–Legendre Nyström and a symmetric piecewise-constant Galerkin
operator. Production runs use triangular cell averages split at every
`r=log(n)`; naive point sampling can violate the contraction bound.

`cusp_galerkin.py` is the focused convergence implementation. It clips each
tensor cell into analytic polygons along `u+v=log(n)`, performs high-order
triangle quadrature, and exploits the translated Hankel blocks. The closure
results are `focused_fredholm_convergence.csv`,
`focused_operator_stability.csv`, `focused_refinement.csv`, and
`focused_compression_rank.csv`.

`dip_binding.cpp` compiles a C ABI around the repository DIP kernel. The
convolution length is padded to the next power of two at least as large as the
linear-convolution length. `dip_hankel.py` reflects the vector, convolves, and
crops indices `[N-1,2N-1)`.

Packet instrumentation has two layers:

- `dip_instrumentation.cpp` emits the production kernel's `(d,e)` topology and
  exact phase-table labels without altering arithmetic.
- `dip_packet_observer.py` mirrors the documented shipped cell equations to
  capture energies and exact local roundtrip residuals. The representative
  observations are in `data/dip_packet_observables.csv`.

The packet observer is an experimental observable, not an FFT correctness test.

## Reproduce

Requirements already present in the local environment are Python 3, NumPy,
SciPy, mpmath, Matplotlib, and Clang. The checked-in Parquet artifact was emitted with PyArrow; rerunning without
PyArrow preserves a CSV fallback and reports that fact explicitly.

```bash
python3 scripts/run_baseline.py
python3 scripts/run_omega_sweep.py
python3 scripts/run_focused_closure.py
```

Use `python3 scripts/run_baseline.py --full` to request the handoff's full
`N=64..1024` grid. It is intentionally not represented as already certified.

All requested sources, scripts, CSV/Parquet data, and seven plots are present
