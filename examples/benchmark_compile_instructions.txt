in bfft folder:

First install FFTW systemwide if you want the dynamic FFTW comparison columns.

Optional PFFFT build for the benchmark:

git clone https://github.com/marton78/pffft.git pffft

mkdir -p build/pffft build/examples

cc -O3 -DNDEBUG \
  -Ipffft/include/pffft \
  -Ipffft/src \
  -c pffft/src/pffft.c \
  -o build/pffft/pffft.o

cc -O3 -DNDEBUG \
  -Ipffft/include/pffft \
  -Ipffft/src \
  -c pffft/src/pffft_common.c \
  -o build/pffft/pffft_common.o

cc -O3 -DNDEBUG \
  -Ipffft/include/pffft \
  -Ipffft/src \
  -c pffft/src/fftpack.c \
  -o build/pffft/fftpack.o

Then link benchmark:

c++ -O3 -std=c++17 -DNDEBUG -DBFFT_WITH_PFFFT \
  -Iinclude \
  -Ipffft/include/pffft \
  -Ipffft/src \
  -I/opt/homebrew/include \
  examples/benchmark.cpp \
  build/libbfft.a \
  build/pffft/pffft.o \
  build/pffft/pffft_common.o \
  build/pffft/fftpack.o \
  -L/opt/homebrew/lib \
  -lfftw3 \
  -lm \
  -ldl \
  -o build/examples/benchmark

Optional Intel oneMKL comparison:

Install a package that provides libmkl_rt.so, such as Intel oneAPI MKL or a
system Intel MKL package. The benchmark does not link MKL at build time. It
loads libmkl_rt dynamically only when you pass --intel-mkl, so the same binary
still runs on machines without MKL.

Then:

build/examples/benchmark --intel-mkl
build/examples/benchmark --intel-mkl 1024 1000

Locality probe:

The ordinary make examples target also builds build/examples/locality_probe.
It compares hot-cache timing with a rotating working set sized by ring_mib.

build/examples/locality_probe
build/examples/locality_probe 65536 240 64
