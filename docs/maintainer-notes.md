# Maintainer notes for GitHub repository settings

These changes cannot be applied from inside the working tree. Please configure
these in the GitHub UI when you prepare the repository for public release.

## Repository appearance

- Add a repository description: `Bruun real FFT library for C and C++`.
- Add topics: `fft`, `dsp`, `c`, `cpp`, `simd`, `bruun-transform`.
- Set the website field to the hosted documentation URL once available.
- Pin or highlight the `examples/benchmark.cpp` demo in release notes.

## Branch protection

- Protect the default branch.
- Require pull requests before merging.
- Require `make test` in CI before merging once CI is added.
- Require linear history if that matches the maintainer workflow.

## Release assets

- Publish source archives generated by GitHub releases.
- Attach prebuilt packages only after platform-specific CI exists.
- Include the MIT license in all release artifacts.

## Future CI

Add matrix jobs for:

- Linux GCC and Clang.
- macOS Clang.
- Windows MSVC and MinGW.
- x86_64 AVX2-capable runners and scalar fallback builds.
- aarch64 NEON runners when available.
